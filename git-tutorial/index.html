Create a new repository

Open your terminal and type:

mkdir ~/myNewRepository && cd ~/myNewRepository

Now you're in the folder, so let's start the repository

git init

Work Flow

Your local repository is composed of three "trees" managed by git. The first is your Working Directory that contains the files, the second is the Index that acts as an intermediate zone, and the last one is the HEAD that points to the last commit made.

Checkout repository

Create a local copy of the repository using

git clone /path/to/repository

If you're using a remote server:

git clone username@host:/path/to/repository

Add & Commit

You can register changes (add them to the Index) using

git add <filename>

or to add everything

git add .

This is the first step in the basic workflow. To commit these changes, use

git commit -m "Commit message"

Now the file is included in the HEAD, but not yet in your remote repository.

Send changes

Your changes are now in the HEAD of your local copy. To send these changes to your remote repository run

git push origin master

Replace master by the branch to which you want to send your changes.

If you have not cloned an existing repository and want to connect your local repository to a remote repository, use

git remote add origin <server>

Now you can upload your changes to the selected remote repository.

Branches

The branches are used to develop functionalities isolated from each other. The master branch is the "default" branch when you create a repository. Create new branches during development and merge them to the main branch when you finish.
